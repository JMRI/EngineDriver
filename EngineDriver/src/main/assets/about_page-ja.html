この無料アプリケーションは、コンピュータ上で実行されている JMRI WiThrottle サーバー、MRC Prodigy WiFi、または Digitrax LnWi に接続し、機関車と鉄道模型全体を制御できます。速度、方向、および最大 32 の DCC 機能が 1 ～ 6 台の機関車または編成でサポートされています。 Web ビューを使用して、レイアウトの電源、分岐器、ルートを制御し、パネルやその他の JMRI ウィンドウを表示できます。構成 (ソフトウェア定義) を作成および編集することもできます。

前提条件 (JMRI の場合):

Android デバイスが WiFi 経由でネットワークに接続し、IP アドレスを取得できることを確認します。
PC で最新バージョンの JMRI (2.8 で動作、3.0 以上で動作) のいずれかを実行し、コンピュータに接続していることを確認してください。レイアウト。  JMRI を起動し、付属の「Throttle」アプリを使用してレイアウト上の機関車を制御できることを確認します。

DecoderPro の [ツール]、[スロットル] の下にある JMRI の WiThrottle 機能を開始します。また、JMRI Web サーバー機能を開始することを強くお勧めします ([ツール]、[JMRI Web サーバーの開始])。

これらは両方とも、[環境設定]、[スタートアップ] にも追加する必要があります。

前提条件 (MRC Wifi の場合):

MRC WiFi モジュールを接続し、デバイスがネットワークに接続して IP アドレスを取得できることを確認します。

前提条件 (Digitrax LnWi の場合):

LnWi に接続し、デバイスがネットワークに接続して IP アドレスを取得できることを確認します。

ネイティブ DCC-EX コマンドを使用する場合、追加のメニュー項目 (DCC-EX) が利用可能になります。 CV やその他の機能のプログラミング。

手術：

携帯電話で Engine Driver アプリを起動し、サーバーのアドレスが「検出されたサービス」に表示されるまで待ちます。

検出されたアドレスをクリックするか、WiThrottle サーバーのアドレスとポートを直接入力できます。 (WiThrottle ウィンドウに表示される値を使用します。)
「最初の WiThrottle サーバーに自動接続しますか?」というメッセージがあります。必要に応じて有効にすることができます。 jmri.mstevetodd.com のポート 44444 には、テストのために接続できる「デモ」サーバーもあります。

スロットル画面で、各機関車の「選択」機関車ボタンをクリックします。
次の画面で、選択方法 (DCC アドレス入力、JMRI 名簿、または最近の機関車) を選択します。機関車を選択すると、元の画面に戻ります。
さらに多くの列車を取得するには、追加の「選択」ボタンを使用して繰り返します。
注: 名簿項目を長押しすると、名簿から追加の詳細が表示されます。

スライダー (またはボリューム コントロール) (またはオプションの矢印ボタン) を使用して速度を調整し、後進、停止、前進ボタンを使用して方向を制御して機関車を操作します。
すべての機関車を解放するには、[戻る] ボタンを押してアプリを終了します。これにより、機関車も停止します (設定で上書きできます)。速度インジケーターをクリックしてボリューム コントロールを再割り当てできます (小さな「v」は、どちらがコントロールしているかを示します)。

「オンザフライ」構成を作成するには、同じスロットルに対して追加の機関車を選択するだけです。既存の編成に対する新しい各機関車の方向を指定するよう求められます。方向と速度は全員に送信されます。 Release は、そのスロットルのすべてを解放します。

携帯電話のメニュー ボタンを押して、出動、ルート、電力制御の画面にアクセスしたり、設定を調整したり、さまざまな設定を行ったりできます。

また、左または右に「フリック」して、スロットル、ルート、および分岐器のページ間をすばやく移動することもできます。

ESU MobileControl II の操作:

Engine Driver アプリは、ESU MobileControl II Android スロットルを完全にサポートしています。

コントロールノブで選択したスロットルの速度を制御できるだけでなく、ゼロ位置エンドストップスイッチを介して方向を切り替えることもできます。

最大 3 台の列車を同時に制御できます。制御ノブで操作されている列車は、速度インジケーターの小さな「v」で強調表示されます。適切な速度インジケーターをクリックして、運行する列車を切り替えます。デバイスのサイド ボタンの 1 つを、次に運行する列車に切り替えるために割り当てることもできます。デフォルトでは、右下のボタンがこの機能に割り当てられています。

列車の制御に割り当てられている場合、緑色の LED が点灯します。列車が割り当てられていない場合は、緑色の LED が点滅します。

さらに、「停止」ボタンを使用して、現在運行中の列車を一時停止したり、制御されているすべての列車を停止したりできます。停止ボタンを短く押すと (デフォルトでは 1/2 秒未満)、現在制御されている列車の運行が一時停止されます。これを示すために赤色 LED が素早く点滅します。停止ボタンをもう一度押すと、現在運行中の列車が以前の設定速度で再開します。停止ボタンを長押し (デフォルト、1/2 秒以上) と、制御されているすべての列車が停止します。赤色の LED がゆっくり点滅してこれを示します。停止ボタンを 2 回押すと、制御されているすべての列車は停止したままになり、現在選択されている列車を再び制御できるようになります。

停止モード中は、現在停止しているスロットルの画面上の速度変更ボタンとバーは無効になります。制御スロットル間の切り替えも無効になります。

デバイスのサイドボタンを設定して、さまざまなアクションを実行することができます。デフォルトの割り当ては次のとおりです。

  • 左上 - 速度を上げる
  • 左下 - 速度を下げる
  • 右上 - 方向を切り替えます
  • 右下 - 次のスロットルを選択します

これらの割り当ては、[環境設定] の [ESU MobileControl II オプション] ページから変更できます。

構成：

[環境設定] には、デバイス、レイアウト、個人のスタイルに合わせて最適に動作するように EngineDriver の画面とアクションをカスタマイズできるオプションの広範なリストがあります。これらには、スロットル レイアウト、高速クロック表示、速度ステップ オプション、Web ビュー、ボタン サイズ、追加ボタン、スライダーのサイズと動作、表示するスロットルの数、リリース時の停止、オンザフライ構成の許可/禁止、自動接続、およびもっとたくさん。

サーバー内の列車情報エントリにファンクションラベルを入力した場合は、これらのラベルが使用されます。入力していない場合は、[ファンクションデフォルト] メニュー オプションを使用して、表示するデフォルトのラベルとファンクション番号を設定できます。ラベルを空白にしてボタンを削除します。

JMRI コンピュータ側で、WiThrottle の設定をチェックして、必要な制御機能を許可または禁止していることを確認します。パネルで分岐器とルートを定義した場合は、スロットルにリスト表示する分岐器とルートの「ユーザー名」を必ず指定してください。また、WiThrottle ウィンドウの [WiThrottle] -> [コントロールのフィルター] オプションを使用して、表示される分岐器を「フィルター」することもできます。[eStop の使用] を有効にすることも推奨されます。

JMRI で列車情報エントリを入力し、ルートと分岐を定義すると、Engine Driver スロットルがより強力になり、使いやすくなります (必須ではありません)。EngineDriver は、JMRI Web サーバーが起動すると、追加情報 (列車情報の詳細やアイコン画像など) を表示します。
